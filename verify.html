ตรวจสอบบล็อกเชนทั้งระบบ แบบเต็ม

หน้านี้จะย้อนตรวจ ผลรางวัลและอันดับทุกงวด ตั้งแต่งวดแรกที่ระบบนี้ดำเนินการมาจนถึงงวดล่าสุด ว่ายังตรงกับผลที่ประกาศไว้ในงวดนั้น ๆ ที่ผ่านมาหรือไม่

ถ้ามีใครแอบแก้ผลภายหลัง ผลจากการตรวจในหน้านี้ จะขึ้นเตือนให้เห็นทันที

รายละเอียดเชิงเทคนิคแบบเต็ม (สำหรับนักพัฒนา / auditor)

1. สถาปัตยกรรม (Architecture)

Trust Ledger เป็น append-only hash chain — "ผลรางวัลที่ประกาศ 1 เกม = 1 block" (n2/n3 แยก block เชนเดียวทั้งระบบ) · Genesis block #0 มี previousBlockHash = "0"×64

2. Canonical Serialization

ก่อน hash/sign ทุก block ถูก serialize เป็น deterministic canonical JSON: sort object keys แบบ recursive (lexicographic) · ไม่มี whitespace · timestamp เป็น ISO-8601 UTC · field set คงที่ 14 ตัว (เติม null ที่ไม่ใช้ เพื่อกัน ambiguity):

blockNumber, blockType, seasonId, drawId, gameType, winningNumber, winningZone, totalPredictions, predictionMerkleRoot, leaderboardRoot, previousBlockHash, correctsBlockNumber, correctionReason, createdAt

3. Block Hash & Digital Signature

blockHash = SHA-256( canonical(block \ {blockHash, serverSignature}) ) → hex 64 ตัว

serverSignature = Ed25519.sign( canonical เดียวกัน ) → base64 · private key อยู่ใน Secret Manager ฝั่งเซิร์ฟเวอร์ ไม่เคยออกนอกระบบ

Trust-root: หน้านี้ ไม่ได้ ใช้ public key จาก endpoint เป็นหลัก (endpoint คือ เซิร์ฟเวอร์ที่เรากำลังตรวจ — ถ้าถูกยึด ก็ประกาศ key ปลอมได้) แต่โหลดจาก keys.json ที่ pin ไว้ในรีโปสาธารณะนี้ (ย้อนดูประวัติการแก้ไขได้บน GitHub) · รองรับหลาย key ตามช่วงบล็อก (validFromBlock/validToBlock) และสถานะ revoked · ถ้าโหลด keys.json ไม่ได้ = ปฏิเสธการตรวจ (fail-closed) ไม่ปล่อยผ่านเงียบ ๆ · (GET /ledgerPublic?key=1 ยังมีไว้ให้เทียบ แต่ไม่ใช่สิ่งที่หน้านี้เชื่อ)

4. Merkle Commitments

predictionMerkleRoot = binary Merkle root ของคำทายทั้งงวด (leaf = SHA-256(canonical{drawId, gameType, selectedZone, submittedAtMs, uid}) · เรียงตาม uid · จำนวนคี่ = ทำซ้ำใบสุดท้าย) → commit ว่าคำทายชุดใดถูกนับ โดยไม่ต้องเปิดเผยทั้งหมด

leaderboardRoot = Merkle root ของ ตารางอันดับระดับประเทศ ของเกมนั้น ณ ตอนปิดงวด (leaf = SHA-256(canonical{accuracyRate, bestStreak, currentStreak, displayName, rank, totalWins}) · เรียงตาม rank · จำนวนคี่ = ทำซ้ำใบสุดท้าย) → commit อันดับลงเชน = แก้ตารางอันดับในฐานข้อมูลแล้วเนียนไม่ได้

5. Hash Chain → Tamper-Evidence

block[N].previousBlockHash == block[N-1].blockHash · แก้ block ใดย้อนหลังแม้แต่บิตเดียว → hash ของ block นั้นและทุก block หลังจากนั้นเพี้ยน + ลายเซ็นพัง → จับได้ทันที

เป็น tamper-evident ไม่ใช่ tamper-proof: แก้ได้แต่ "ซ่อนไม่ได้" — แม้แต่ผู้ดูแลระบบเองก็แก้ประวัติเงียบๆ ไม่ได้ (ต้องมี private key + ยังพังเชนตั้งแต่จุดที่แก้)

6. ขั้นตอนตรวจสอบ (หน้านี้ทำอะไร)

1) โหลด keys.json (trust-root ที่ pin ในรีโปนี้ — ไม่ใช่จาก endpoint) · 2) ดึงทุก block ผ่าน endpoint สาธารณะ ledgerPublic · 3) คำนวณ SHA-256 ซ้ำ เทียบ blockHash · 4) Ed25519.verify ลายเซ็นด้วย key จาก trust-root (บล็อกไม่เซ็น / verify ไม่ผ่าน = ถือว่าไม่ผ่าน ไม่ downgrade เงียบ) · 5) ตรวจ previousBlockHash linkage ตลอดเชน · 6) ตรวจตารางอันดับ: ดึงตารางอันดับระดับประเทศ ปัจจุบัน ทั้งสองเกม (ledgerPublic?board=n2|n3) มาคำนวณ Merkle ซ้ำ แล้วเทียบกับ leaderboardRoot ที่เซ็นไว้ใน block ผลงวดล่าสุดของเกมนั้น — ไม่ตรง = ตารางถูกแก้ · ทั้งหมดรันในเบราว์เซอร์ของคุณผ่าน Web Crypto API · zero dependency

พารามิเตอร์ที่ใช้ได้: ?block=N ตรวจเฉพาะบล็อกเดียว · ?full=1 เปิดแล้วตรวจทั้งเชนอัตโนมัติ (ปุ่ม "ตรวจสอบเต็ม" ในแอปใช้แบบนี้) · ?allowUnsigned=1 โหมดดีบั๊ก ตรวจแค่ hash + เชน (ไม่ตรวจลายเซ็น)

7. สิ่งที่หน้านี้ตรวจให้คุณ

ทุกครั้งที่กดตรวจสอบ หน้านี้จะดึงข้อมูลสาธารณะมาคำนวณและเทียบใหม่ทั้งหมด ในเบราว์เซอร์ของผู้ตรวจเอง:

· รหัสประจำบล็อก (blockHash) — คำนวณ SHA-256 ซ้ำจาก เนื้อหาบล็อก เทียบกับรหัสที่ประกาศไว้ ครบทุกบล็อกตั้งแต่บล็อกกำเนิด

· ลายเซ็นดิจิทัล (Ed25519) — ตรวจทุกบล็อกด้วยกุญแจสาธารณะจาก trust-root ที่ pin ไว้ในรีโปนี้ (บล็อกไม่เซ็น / verify ไม่ผ่าน = ถือว่าไม่ผ่าน)

· ความต่อเนื่องของเชน — previousBlockHash ของทุกบล็อกต้องตรงกับ รหัสของบล็อกก่อนหน้า ตลอดสายจนถึงบล็อกล่าสุด

· ตารางอันดับ (leaderboardRoot) — ดึงตารางอันดับระดับประเทศปัจจุบัน ทั้งสองเกมมาคำนวณ Merkle ซ้ำ เทียบกับรากที่เซ็นไว้ในบล็อกผลงวดล่าสุดของเกมนั้น
